# 代码规范

本项目源码遵循以下约定，提交代码前请保持一致。

## 1. 模块文件头

每个 `.py` 文件顶部用**模块 docstring** 说明用途，格式：

```python
"""文件名.py — 一句话职责

可选的补充说明（多行）。
"""
import ...
```

不要用 `# 文件名 — ...` 注释行作文件头（已统一为 docstring）。

## 2. 注释

- 注释一律用**中文**，解释「为什么这么做」而非复述代码。
- 不写开发过程的口语化/情绪化注释（如「核心魔法」「杀手锏」「保持不变」「修改点」等），也不在注释或 `print` 里使用 emoji。
- 控制台输出用中性前缀标签，如 `print("[训练] ...")`、`print("[Pass 1] ...")`。
- 类与公有函数尽量写 docstring，说明输入/输出维度与关键假设。

## 3. 命名与格式

- 缩进 4 空格；遵循 PEP 8 的大体风格。
- 模块内私有函数/常量用前缀 `_`（如 `_build_pose_vec`、`_MEAN`）。
- import 顺序：标准库 → 第三方 → 本项目模块。

## 4. 路径与跨平台

- **不要写死绝对路径**（尤其本机路径）。脚本默认从仓库根目录运行，使用相对路径或 `argparse` 参数。
- 大文件目录（`videos/`、`data/`、`models/`、`runs/`）由 `.gitignore` 排除，不入库。
- 处理中文路径时，Windows 用短路径规避 OpenCV 编码问题，非 Windows 必须能跳过：

```python
def _get_short_path(path):
    buf = ctypes.create_unicode_buffer(512)
    if not hasattr(ctypes, "windll"):   # 非 Windows 直接用原路径
        return path
    ctypes.windll.kernel32.GetShortPathNameW(path, buf, 512)
    return buf.value or path
```

## 5. 配置

- 训练超参放 `configs/*.yaml`，不要硬编码在脚本里。
- 新增配置时同步更新 `configs/CONFIG_REFERENCE.md` 与 `configs/README.md`。

## 6. 数据格式约定

- 动作标注：`annotations.json`，动作类别 `待机(0) 正手(1) 反手(2) 发球(3) 移动(4)`。
- 姿态数据：`pose_data.json`，每帧含 `court`(14点) / `near_player` / `far_player`。
- 详见根目录 README 的「数据格式」一节。
